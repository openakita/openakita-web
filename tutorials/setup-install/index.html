教程概览

推荐路径：Desktop 快速启动，CLI 稳定部署，源码模式用于团队二次开发。

1

完成安装

根据角色选择 Desktop / PyPI / 源码方式。

2

执行初始化

运行 `openakita init` 生成 `.env` 与必要目录。

3

做首次验证

确认对话、任务执行与日志输出全部正常。

方式一：Desktop（按配置向导）

以下步骤依据桌面端配置向导整理，覆盖快速配置与完整配置两种模式。首次启动 OpenAkita Desktop 后会自动进入该向导。

开始前请准备：可联网环境、至少 1 个可用 LLM API Key、2 GB 以上可用磁盘空间。

0. 启动与前置检查

项目 建议值 说明
操作系统 Windows 10/11、macOS 12+、Linux x86_64 桌面安装包按系统分别下载
网络 可访问 Python 依赖源和 LLM 服务 首次配置会下载运行环境与依赖
LLM API 至少 1 个可用端点 快速配置必须先添加端点才能继续
配图位：首次启动主界面 建议放 Desktop 启动后自动进入配置向导的页面

1. 选择配置模式

模式 耗时 适合场景 特点
快速配置 约 3 分钟 首次体验、默认部署 自动完成环境搭建，只需填写关键项
完整配置 约 10 分钟 生产定制、企业部署 逐项配置工作区、Python、端点、工具与系统参数
配图位：模式选择页 建议放快速配置 / 完整配置两张卡片
配图位：切换模式入口 建议放侧边栏中的切换配置模式按钮

2. 快速配置（约 3 分钟）

快速配置会自动创建默认工作区、安装内置 Python、创建虚拟环境并安装依赖。你只需补全 LLM 端点，IM 通道可选。

1

添加 LLM 端点（必填）

点击“+ 添加端点”，填写服务商、Base URL、API Key、模型、能力标签。

2

配置 IM 通道（可选）

按需填写 Telegram / 飞书 / 企业微信 / 钉钉 / QQ 官方机器人 / OneBot 参数，不配也可先使用桌面对话。

3

点击开始配置

未添加任何 LLM 端点时按钮会禁用，先补全端点后再继续。

4

等待自动配置完成

向导会按顺序安装运行环境并写入默认配置，可在进度页查看日志。

5

完成并启动服务

在完成页查看工作区、端点和 IM 摘要，点击“启动服务”进入状态面板。

配图位：快速配置填写页 建议放 LLM 端点对话框（含高级选项展开）
配图位：IM 通道填写页 建议放 IM 区域展开后的配置表单
配图位：自动配置进度页 建议放创建工作区、安装 Python、安装依赖、写入配置的分步进度
  • 自动步骤包含：创建默认工作区、安装内置 Python 3.11、创建 venv、安装 openakita[all]、写入默认配置。
  • 网络异常导致失败时，可返回模式选择切换到完整配置进行手动排查。
  • 完成后优先验证一次模型连通性，再继续接入 IM 通道。

3. 完整配置（约 10 分钟）

完整配置适合生产场景，适合需要自定义运行环境、安装源、工具能力和 Agent 行为策略的团队。

4.1 工作区

新建或切换工作区，隔离 .env、data/llm_endpoints.json 和角色配置。

4.2 Python 环境

可选内置 Python（推荐）或系统 Python 3.11+，检测通过后再进入安装步骤。

4.3 安装

支持 PyPI / GitHub / 本地三种来源，可选择镜像源与 extras 组件。

4.4 LLM 端点

支持多端点、优先级和故障切换；主端点建议配置 2 个不同服务商。

4.5 IM 通道

按需启用 Telegram、飞书、企业微信、钉钉、QQ 官方机器人与 OneBot，全部为可选项。

4.6 工具与技能

配置 MCP 工具、桌面自动化、语音与代理参数，并管理内置/外部 Skills。

4.7 Agent 与系统

设置角色人格、日志、记忆、主动消息与调度器参数。

4.8 完成

检查配置文件、启动服务并进入状态面板，必要时可执行清理或重装。

安装来源 说明 推荐场景

GitHub 拉取最新仓库代码 需要新特性或测试分支
本地目录 从本地源码安装 本地开发和调试

4. 配置完成后的文件位置

向导会把配置写入工作区目录。建议通过桌面端修改配置，避免手工编辑导致格式错误。

# 环境变量文件 ~/.openakita/workspaces/{workspace_id}/.env # LLM 端点列表 ~/.openakita/workspaces/{workspace_id}/data/llm_endpoints.json # 角色灵魂文件 ~/.openakita/workspaces/{workspace_id}/identity/SOUL.md
  • 修改 .env 或 llm_endpoints.json 后，执行“应用并重启”或重启服务。
  • 生产环境建议保留一份可回滚的配置快照。

方式二：PyPI CLI

适合研发环境和自动化部署脚本，推荐作为 CI/CD 的基础安装路径。

# 1) 创建虚拟环境 python -m venv venv # 2) 激活环境（Linux/macOS） source venv/bin/activate # 3) 激活环境（Windows PowerShell） # .\\venv\\Scripts\\activate # 4) 安装 openakita 与常用扩展 pip install openakita[all] # 5) 初始化配置文件 openakita init # 6) 启动交互模式 openakita

按需安装可选包：`openakita[feishu]`、`openakita[whisper]`、`openakita[browser]` 等。

配图位：命令行安装截图建议放 pip 安装和 init 成功输出

方式三：源码安装

适合研发团队进行二次开发、定制模型策略和平台适配器扩展。

# 1) 拉取源码 git clone https://github.com/openakita/openakita.git cd openakita # 2) 创建并激活虚拟环境 python -m venv venv source venv/bin/activate # 3) 升级 pip 并安装开发依赖 pip install --upgrade pip pip install -e ".[all,dev]" # 4) 准备配置模板 cp examples/.env.example .env cp data/llm_endpoints.json.example data/llm_endpoints.json # 5) 初始化运行环境 openakita init
  • 确认 `openakita` 命令可执行。
  • 确认 `data/llm_endpoints.json` 存在且至少配置一个端点。
  • 确认 `.env` 中至少有一个有效 API Key。

首次启动验证

# 交互模式 openakita # 服务模式（仅 IM） openakita serve # 单次任务测试 openakita run "创建一个带测试的 Python 计算器"

上线前检查清单

  • 对话请求可返回，日志无连续 ERROR。
  • 模型调用成功，响应延迟在可接受范围。
  • 配置文件已备份（`.env`、`llm_endpoints.json`）。
  • 若接 IM 通道，平台回调或长连接已验证通过。

Q：提示 API key not found？

A：检查 `.env` 路径与变量名，确认与 `api_key_env` 一致。

Q：安装成功但命令不可用？

A：确认虚拟环境已激活，或使用 `python -m openakita` 方式启动。

Q：大陆网络调用超时？

A：配置 `ALL_PROXY` 或改用可访问的 `base_url` 代理端点。
